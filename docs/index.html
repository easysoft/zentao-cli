Zentao CLI

把禅道能力带进终端，也带进 AI Agent 的工作流。

Key Features

这不是一个只会“列列表”的薄封装，而是一套同时面向日常操作与 AI 协作的禅道入口：安装更快、登录更稳、筛选更准、接入更顺。

API 2.0

基于最新 RESTful API 2.0

围绕禅道开放接口组织能力，适合长期维护与扩展，也方便对模块能力做统一封装。

Zero Setup

可安装，也可免安装运行

支持 npm install -g zentao-cli 供长期使用，也支持 npx zentao-cli、pnpm dlx zentao-cli 快速体验。

Profiles

安全认证与多账号切换

支持登录多个禅道服务和账号，登录后保存 Token，不保存密码，并可用 zentao profile 快速切换。

Data Ops

筛选、排序、摘取字段更顺手

通过 --pick、--filter、--sort、--search、--limit 等参数，把返回结果压缩成真正想要的数据。

AI Friendly

支持 Markdown 输出与 HTML 转换

默认输出更适合阅读和粘贴给 Agent；同时会自动将 HTML 字段转换为 Markdown，降低二次处理成本。

Modern Stack

Bun + TypeScript + 测试覆盖

使用现代工具链实现，具备类型安全与测试保障，既适合作为 CLI 工具，也适合接入自动化场景。

Three Paths

同一套禅道能力，可以按你的工作方式来用。你可以在终端手动执行，也可以让 Agent 通过 Skill 或 MCP 来调用。

01 作为命令行工具

适合开发、测试、实施、项目经理直接上手

如果你习惯在终端里做查询、更新和批量处理，这条路径最直接。登录一次后，日常执行 zentao 即可。

  • 支持列表、详情、创建、更新、删除，以及模块动作调用。
  • 支持简写方式 zentao product，也支持原始方式 zentao ls product。
  • 适合脚本集成、终端操作、CI 辅助任务。
02 作为 Agent 技能

适合让 AI 直接理解你的业务问题

通过 zentao add-skill，可以一键把能力装进 Agent 工具，让 AI 直接按自然语言访问禅道数据，无需你手动组织复杂命令。

  • 目前支持 Claude Code、Cursor、Cherry Studio、Codex、OpenCode、VS Code 等工具。
  • 安装后可以直接提问，例如“产品 XXX 有哪些研发中的需求？”
  • 适合调研、汇总、问答和让 Agent 协助分析项目状态。
03 作为 MCP 服务

适合标准化接入多种 AI 客户端

通过 zentao add-mcp 可一键生成配置，也支持用 npx -y zentao-cli mcp 手动启动服务，接入各类支持 MCP 的客户端。

  • 客户端无需预装 zentao-cli，本地只要能执行 npx 即可。
  • 通过环境变量传入 URL、账号和密码即可完成运行时认证。
  • 适合桌面 Agent、统一团队配置和更通用的工具集成方式。

Choose Your Flow

同一套能力，按不同接入方式使用。左侧先给你决策建议，右侧直接展示 README 中的典型命令与配置示例。

CLI 当你要自己执行命令、写脚本、做批量处理时，它最快。
Skill 当你希望 Agent 通过自然语言帮你查询和总结禅道信息时，它最轻量。
MCP 当你要把禅道能力标准化接入多个客户端时，它最稳妥。
README 示例节选 可复制到你的终端 / 客户端配置 
# 安装或免安装运行
npm install -g zentao-cli
# 或：npx zentao-cli

# 首次登录
zentao login -s https://zentao.example.com -u admin -p 123456

# 查看产品
zentao product

# 摘取字段、筛选、排序
zentao product --pick=id,name
zentao product --filter 'name~平台'
zentao product --sort=name_asc

# JSON 输出，适合程序消费
zentao bug --format=json
# 一键安装到 AI Agent
zentao add-skill

# 也可以指定目标
zentao add-skill claude-code

# 从零开始的一键流程
pnpm install -g zentao-cli && zentao login && zentao add-skill --all

# 安装完成后，在 Agent 中可以直接这样提问：
禅道中有哪些产品？
产品 XXX 有哪些研发中的需求？
需求 XXX 有哪些风险？
# 一键配置 MCP
zentao add-mcp

# 手动启动服务
npx -y zentao-cli mcp

Typical Workflow

不管你选择哪条路径，Zentao CLI 的核心流程都很清晰：连接禅道、抽取数据、交给人或 Agent 分析，再把结果回写。

01

登录与配置

首次运行支持交互式登录，也支持通过命令参数或环境变量登录，并可保存多套账号配置。

02

查询与筛选

通过模块命令获取列表或详情，再用 --pick、--filter、--sort 聚焦出真正有价值的信息。

03

分析与协作

结果可直接在终端阅读，也可输出 JSON、Markdown 供脚本或 Agent 继续处理与总结。

04

更新与自动化

再通过创建、更新、删除和动作命令回写禅道，或借助 Skill / MCP 接入更长链路的自动化流程。

Why Agents Like It

Zentao CLI 并不是“顺带能给 Agent 用”，而是有意识地把输出、帮助、接入方式都做成了适合 AI 消费的形态。

Readable Output

输出天然适合二次理解

默认 Markdown 更适合人读，也更适合 Agent 复制、总结和继续生成结构化结论。

  • HTML 字段自动转 Markdown，减少清洗工作。
  • 支持 json 和 raw，方便程序链路处理。
Explicit Help

帮助信息更完整，减少试错

Agent 不需要猜测命令结构，可以直接从帮助信息中稳定找到模块、参数与调用方式。

  • 支持 zentao help 与 zentao <module> help。
  • 同时支持简写和原始调用方式，兼顾人和工具。
One-Click Install

接入 Agent 的门槛足够低

通过 add-skill 与 add-mcp，把繁琐配置压缩成可执行命令，让个人和团队都能更快落地。

  • 支持多个常见 Agent 平台。
  • 可从零开始完成安装、登录与接入。
Composable

适合被放进更长的自动化链路

CLI、Skill、MCP 三种模式共享同一能力底座，既适合手工操作，也适合持续接入自动化任务。

  • 本地终端、桌面客户端、开发工具都能接入。
  • 既能查，也能改，适合闭环工作流。

Start Fast

如果你想先把它跑起来，再决定是否接入 Agent，这组命令已经足够开始。

长期使用建议全局安装；如果只想快速体验，直接用 npx 即可。后续如果你希望让 Agent 代劳，再继续执行 zentao add-skill 或 zentao add-mcp 即可。 

# 先安装或直接试跑
npm install -g zentao-cli
# 或：npx zentao-cli

# 登录后开始查询
zentao login
zentao help
zentao product
